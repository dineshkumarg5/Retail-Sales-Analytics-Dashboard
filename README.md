## рҹ“Ң Project Overview

This project showcases a **production-quality retail sales analytics pipeline** built on the **Superstore Sales Dataset** вҖ” a realistic retail dataset with 9,994 transactions spanning 4 years (2014вҖ“2017) across the United States.

The project demonstrates the **complete Data Analyst workflow**:

```
Raw Data вҶ’ Cleaning вҶ’ EDA вҶ’ SQL Analysis вҶ’ Visualization вҶ’ Insights вҶ’ Dashboard
```

It is designed to reflect the kind of work done in entry-level to junior Data Analyst, Business Analyst, and Data Science roles вҖ” not a tutorial, but a real portfolio deliverable.

### Customer Segmentation (RFM)
Using **Recency, Frequency, Monetary** scoring, customers were classified into:
- рҹҸҶ **Champions** вҖ” Most recent, frequent, and high-value buyers
- рҹ’Һ **Loyal Customers** вҖ” Consistent, high-value, slightly older purchases
- рҹҢұ **Potential Loyalists** вҖ” Recent buyers with growth potential
- вҡ пёҸ **At Risk** вҖ” Previously good customers who've gone quiet
- рҹҳҙ **Hibernating** вҖ” Low recent activity
- вқҢ **Lost** вҖ” No recent interaction

### Sample Query вҖ” RFM Segmentation in SQL
```sql
WITH customer_rfm AS (
    SELECT
        c.customer_id, c.customer_name, c.segment,
        DATEDIFF('2018-01-01', MAX(o.order_date)) AS recency_days,
        COUNT(DISTINCT o.order_id)                 AS frequency,
        ROUND(SUM(oi.sales), 2)                    AS monetary
    FROM customers c
    JOIN orders o      ON c.customer_id = o.customer_id
    JOIN order_items oi ON o.order_id   = oi.order_id
    GROUP BY c.customer_id, c.customer_name, c.segment
)
SELECT *,
    CASE
        WHEN (r_score + f_score + m_score) = 9 THEN 'Champions'
        WHEN (r_score + f_score + m_score) >= 7 THEN 'Loyal Customers'
        WHEN (r_score + f_score + m_score) >= 5 THEN 'Potential Loyalists'
        ELSE 'At Risk / Lost'
    END AS customer_segment
FROM customer_rfm;
```

## рҹ’Ў Business Insights

### рҹ”ҙ Critical Issues
1. **Discounts above 30%** cause losses on 58% of transactions вҶ’ immediate policy change needed
2. **Furniture Tables** sub-category loses -$17,725 annually вҶ’ reprice or exit
3. **Central Region** has only 7.9% margin вҶ’ investigate cost structure

### рҹҹЎ Growth Opportunities
4. **Corporate segment** has the highest avg order value ($520) вҶ’ dedicate account managers
5. **Technology margins are 17%+** вҶ’ increase marketing budget
6. **30% of customers are At Risk or Hibernating** вҶ’ RFM re-engagement campaigns

### рҹҹў What's Working
7. **Consistent YoY revenue growth** (20%+ in 2017)
8. **West Region** is the most profitable вҖ” use as benchmark for others
9. **Copiers sub-category** has 37% margin вҖ” expand to underperforming regions

> *These insights are based on exploratory data analysis of the Superstore dataset and are intended to demonstrate how analytical findings can support business decision-making.*

## рҹҡҖ How to Run

### Prerequisites
```
Python 3.9+    |    MySQL 8.0+    |    Jupyter Notebook    |    Power BI Desktop
```

### Quick Start
```bash
# 1. Clone the repository
git clone https://github.com/dineshkumarg5/Retail-Sales-Analytics-Dashboard.git
cd Retail-Sales-Analytics-Dashboard

# 2. Install dependencies
pip install -r requirements.txt

# 3. Generate all outputs (clean CSV + 10 charts)
py -3.12 generate_outputs.py

# 4. Launch Jupyter for interactive notebooks
jupyter notebook notebooks/
```

### Run Notebooks in Order
```
01_data_collection.ipynb     вҶ’ Dataset loading & profiling
02_data_cleaning.ipynb       вҶ’ Cleaning, types, feature engineering
03_exploratory_data_analysis вҶ’ 15+ EDA analyses
04_data_visualization.ipynb  вҶ’ 10 professional charts
05_business_insights.ipynb   вҶ’ RFM, trends, recommendations
```

### SQL Setup
```bash
mysql -u root -p < sql/01_ddl_create_tables.sql
mysql -u root -p < sql/02_insert_data.sql
# Continue in numeric order through 09
```
